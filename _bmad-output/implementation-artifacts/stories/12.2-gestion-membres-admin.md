# Story 12.2 : Gestion des membres (admin)

## Description

La page Reglages affiche actuellement la liste des membres avec leurs roles (Admin/Membre) et un bouton "Inviter un colocataire", mais aucune action de gestion n'est possible. L'admin ne peut ni promouvoir, ni retrograder, ni exclure un membre. Cette story ajoute un menu contextuel par membre avec toutes les actions de gestion, et les routes API correspondantes.

## User Story

As a **admin de colocation**,
I want **pouvoir modifier le role d'un membre, le virer de la colocation, ou le promouvoir admin**,
So that **je controle qui fait partie de ma colocation et avec quels droits**.

## Criteres d'Acceptation

**AC-1: Menu contextuel par membre**
**Given** je suis admin sur la page Reglages
**When** je clique sur le bouton menu (trois points) a cote d'un membre
**Then** je vois les options: "Promouvoir Admin", "Retrograder Membre", "Exclure de la colocation"
**And** les options non applicables sont grisees (ex: "Promouvoir" sur un admin, "Retrograder" sur un membre)

**AC-2: Promouvoir un membre en admin**
**Given** je suis admin et je clique "Promouvoir Admin" sur un membre
**When** la confirmation s'affiche et je confirme
**Then** le role du membre passe a "admin"
**And** le badge dans la liste passe de "Membre" a "Admin"

**AC-3: Retrograder un admin en membre**
**Given** je suis admin et il y a au moins 2 admins
**When** je clique "Retrograder Membre" sur un autre admin et confirme
**Then** le role de cet admin passe a "member"
**And** le badge dans la liste passe de "Admin" a "Membre"

**AC-4: Empecher la retrogradation du dernier admin**
**Given** je suis le seul admin de la colocation
**When** je tente de me retrograder
**Then** un message d'erreur s'affiche "Vous devez d'abord promouvoir un autre membre admin"
**And** mon role ne change pas

**AC-5: Exclure un membre**
**Given** je clique "Exclure" sur un membre (non-admin)
**When** la popup de confirmation s'affiche avec le nom du membre et je confirme
**Then** le membre est retire de la colocation
**And** la liste des membres se met a jour
**And** l'utilisateur exclu voit sa `colocationId` mise a null

**AC-6: Pas d'actions sur soi-meme (exclusion)**
**Given** je suis admin
**When** je regarde mon propre menu
**Then** l'option "Exclure de la colocation" n'est pas disponible pour moi-meme

## Taches Techniques

### Task 1: Creer la route PUT /api/colocation/:id/members/:userId (modifier role)

**File:** `server/index.js`

Ajouter apres la route `POST /api/colocation/join` (ligne ~147):

```javascript
app.put('/api/colocation/:id/members/:userId', (req, res) => {
  const { role } = req.body || {};
  if (!role || !['admin', 'member'].includes(role)) {
    return res.status(400).json({ error: 'Role invalide' });
  }
  const userIdx = users.findIndex(u => u.id === req.params.userId);
  if (userIdx === -1) return res.status(404).json({ error: 'Utilisateur non trouve' });
  if (!colocation.members.includes(req.params.userId)) {
    return res.status(404).json({ error: 'Membre non trouve dans la colocation' });
  }
  // Prevent demoting the last admin
  if (role === 'member') {
    const adminCount = users.filter(u =>
      colocation.members.includes(u.id) && u.role === 'admin'
    ).length;
    if (adminCount <= 1 && users[userIdx].role === 'admin') {
      return res.status(400).json({ error: 'Impossible de retrograder le dernier admin' });
    }
  }
  users[userIdx].role = role;
  db.save('users', users);
  res.json({ data: enrichColocation(colocation) });
});
```

### Task 2: Creer la route DELETE /api/colocation/:id/members/:userId (exclure)

**File:** `server/index.js`

```javascript
app.delete('/api/colocation/:id/members/:userId', (req, res) => {
  const memberIdx = colocation.members.indexOf(req.params.userId);
  if (memberIdx === -1) {
    return res.status(404).json({ error: 'Membre non trouve dans la colocation' });
  }
  const userIdx = users.findIndex(u => u.id === req.params.userId);
  // Prevent removing an admin (must demote first)
  if (userIdx !== -1 && users[userIdx].role === 'admin') {
    return res.status(400).json({ error: 'Impossible d\'exclure un admin. Retrogradez-le d\'abord.' });
  }
  // Remove from colocation members
  colocation.members.splice(memberIdx, 1);
  db.save('colocation', colocation);
  // Clear user's colocationId
  if (userIdx !== -1) {
    users[userIdx].colocationId = null;
    db.save('users', users);
  }
  res.json({ data: enrichColocation(colocation) });
});
```

### Task 3: Ajouter les fonctions API client

**File:** `client/src/lib/api.js`

```javascript
export function updateMemberRole(colocationId, userId, role) {
  return request(`/api/colocation/${colocationId}/members/${userId}`, {
    method: "PUT",
    body: JSON.stringify({ role })
  })
}

export function removeMember(colocationId, userId) {
  return request(`/api/colocation/${colocationId}/members/${userId}`, {
    method: "DELETE"
  })
}
```

## Endpoints API

### PUT /api/colocation/:id/members/:userId

**Request Body:**
```json
{ "role": "admin" }
```

**Response 200:**
```json
{
  "data": {
    "id": "coloc-1",
    "name": "Colocation Jopad & Co",
    "invitationCode": "COLO-7829-X",
    "totalFund": 245.50,
    "members": [
      { "id": "user-1", "name": "Thomas Durand", "email": "thomas@coloc.fr", "role": "admin" },
      { "id": "user-2", "name": "Lea Martin", "email": "lea@coloc.fr", "role": "admin" }
    ]
  }
}
```

**Response 400 (dernier admin):**
```json
{ "error": "Impossible de retrograder le dernier admin" }
```

### DELETE /api/colocation/:id/members/:userId

**Response 200:** Colocation enrichie sans le membre exclu

**Response 400 (admin):**
```json
{ "error": "Impossible d'exclure un admin. Retrogradez-le d'abord." }
```
