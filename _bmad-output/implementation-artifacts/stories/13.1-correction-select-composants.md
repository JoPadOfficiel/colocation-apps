# Story 13.1 : Correction globale des composants Select

## User Story

As a **colocataire**,
I want **que tous les selecteurs (assignation, payeur, statut, type) fonctionnent correctement partout**,
So that **je peux remplir les formulaires sans etre bloque**.

## Description

Les composants `Select` bases sur `@base-ui/react` ne fonctionnent pas correctement lorsqu'ils sont places dans un `Dialog`. Le portail du Select s'affiche derriere le Dialog (z-index insuffisant) ou ne se positionne pas correctement. Cela bloque les formulaires de creation/modification de taches et de depenses.

Le composant `client/src/components/ui/select.jsx` utilise `@base-ui/react/select` avec un `SelectPortal` / `SelectPrimitive.Positioner` qui a un `z-[100]`. Le Dialog shadcn/ui utilise typiquement `z-50`. Le probleme est que le positionnement du Positioner dans le portail entre en conflit avec le Dialog overlay.

## Criteres d'Acceptation

**Given** un Select est dans un Dialog (ajout tache, ajout depense, etc.)
**When** je clique pour ouvrir le Select
**Then** les options s'affichent correctement au-dessus du Dialog
**And** je peux selectionner une option et la valeur est enregistree

**Given** un Select affiche des utilisateurs
**When** je l'ouvre
**Then** les noms sont affiches (pas les IDs)

**Given** je selectionne une valeur dans un Select en Dialog
**When** je soumets le formulaire
**Then** la valeur selectionnee est correctement envoyee a l'API

**Given** je suis sur la page Taches en dehors d'un Dialog
**When** j'utilise les filtres Select (statut, assignee, date)
**Then** les filtres fonctionnent normalement (pas de regression)

### Task 2 : Corriger le composant Select pour fonctionner dans un Dialog

**Option A (recommandee) : Augmenter le z-index du Positioner**

Modifier `client/src/components/ui/select.jsx` :

```jsx
// Line 33-36 : Augmenter z-index du Positioner
<SelectPrimitive.Positioner
  data-slot="select-positioner"
  sideOffset={sideOffset}
  className="z-[200]"  // Was z-[100], must be > Dialog overlay z-index
>
```

Et le Popup (line 38-45) :
```jsx
<SelectPrimitive.Popup
  ref={ref}
  data-slot="select-content"
  className={cn(
    "z-[200] min-w-[var(--anchor-width)] ...",  // Was z-[100]
    className
  )}
```

**Option B (fallback si Option A ne suffit pas) : Remplacer par des select natifs dans les Dialog**

Dans `client/src/pages/Tasks.jsx`, remplacer les Select @base-ui dans le Dialog (L328-381) par des `<select>` HTML natifs styles avec Tailwind :

```jsx
// Replace Select for category (L331-341)
<select
  value={form.category}
  onChange={(e) => setForm((f) => ({ ...f, category: e.target.value }))}
  className="w-full h-8 rounded-lg border border-input bg-white px-2.5 py-1 text-sm"
>
  {CATEGORIES.map((c) => <option key={c} value={c}>{c}</option>)}
</select>

// Replace Select for assignedTo (L345-358)
<select
  value={form.assignedTo}
  onChange={(e) => setForm((f) => ({ ...f, assignedTo: e.target.value }))}
  className="w-full h-8 rounded-lg border border-input bg-white px-2.5 py-1 text-sm"
>
  <option value="none">Non assigne</option>
  {users.map((u) => <option key={u.id} value={u.id}>{u.name}</option>)}
</select>

// Replace Select for recurrence (L370-380)
<select
  value={form.recurrence}
  onChange={(e) => setForm((f) => ({ ...f, recurrence: e.target.value }))}
  className="w-full h-8 rounded-lg border border-input bg-white px-2.5 py-1 text-sm"
>
  {RECURRENCES.map((r) => <option key={r.value} value={r.value}>{r.label}</option>)}
</select>
```

Meme chose dans `client/src/pages/Finances.jsx` L655-670 pour le Select "Paye par" :

```jsx
<select

```
