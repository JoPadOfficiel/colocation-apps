# Story 11.4 : Correction du Select "Paye par" dans le Dialog

## User Story

As a **colocataire**,
I want **pouvoir selectionner le payeur dans le formulaire d'ajout de depense**,
So that **la depense est correctement attribuee a la bonne personne**.

## Description

Le composant `Select` utilise dans le Dialog d'ajout/modification de depense (`Finances.jsx` lines 655-670) est base sur `@base-ui/react/select` (via `client/src/components/ui/select.jsx`). Ce composant utilise un portail (`SelectPrimitive.Portal` / `SelectPrimitive.Positioner`) avec un `z-index: z-[100]`.

Le probleme est que lorsque le Select est rendu dans un Dialog shadcn/ui, le portail du Select entre en conflit avec l'overlay du Dialog. Le Positioner peut se retrouver masque derriere le Dialog ou mal positionne, empechant la selection d'une valeur.

Le composant `select.jsx` actuel :
- Line 2 : importe `Select` depuis `@base-ui/react/select`
- Line 29 : utilise `SelectPrimitive.Portal`
- Lines 33-36 : `SelectPrimitive.Positioner` avec `className="z-[100]"`
- Lines 38-48 : `SelectPrimitive.Popup` avec `className="z-[100] ..."`

Le Dialog shadcn/ui utilise un overlay avec `z-50` et un content avec `z-50`. Le z-[100] devrait theoriquement suffire, mais le Positioner de base-ui calcule sa position par rapport au viewport et non par rapport au Dialog, ce qui cause des problemes de positionnement.

**Solution choisie** : Remplacer le Select `@base-ui/react` par un `<select>` HTML natif style avec Tailwind dans le Dialog de Finances.jsx. Cette approche est fiable a 100% dans un Dialog car le `<select>` natif utilise le mecanisme natif du navigateur pour le dropdown, qui est toujours au-dessus de tout.

Cette story est volontairement scopee au Select "Paye par" de Finances. La Story 13.1 couvre la correction globale de tous les Select dans des Dialog.

## Criteres d'Acceptation

**Given** je clique "AJOUTER DEPENSE"
**When** le Dialog s'ouvre et je clique sur le selecteur "Paye par"
**Then** la liste deroulante affiche les noms des membres (pas les IDs)
**And** je peux selectionner un membre et la valeur est correctement enregistree

**Given** je selectionne "Lea Martin" comme payeur
**When** je valide la depense
**Then** la depense est enregistree avec `paidBy: "user-2"` (ID de Lea) et le tableau affiche "Lea Martin"

**Given** j'ouvre le Dialog de modification d'une depense existante
**When** le Dialog s'ouvre
**Then** le Select affiche le nom du payeur actuel (pas son ID)

**Given** je soumets le formulaire sans selectionner de payeur
**When** la validation s'execute
**Then** un message d'erreur s'affiche : "Le payeur est requis"

## Taches Techniques

### Task 1 : Remplacer le Select @base-ui par un select natif dans le Dialog Finances

**Fichier : `client/src/pages/Finances.jsx`**

1. Retirer l'import Select de la ligne 9 (puisqu'on ne l'utilise plus dans ce fichier) :

```javascript
// AVANT (line 9):

// APRES: Supprimer cette ligne entierement
// (Si d'autres Select sont utilises dans ce fichier, garder l'import)
```

2. Remplacer le bloc Select "Paye par" (lines 653-671) par un select natif :

```jsx
{/* AVANT: */}
{/*
<div className="space-y-2">
  <Label htmlFor="paidBy">Paye par</Label>
  <Select
    value={formData.paidBy}
    onValueChange={(value) => setFormData({ ...formData, paidBy: value })}
    required
  >
    <SelectTrigger id="paidBy">
      <SelectValue placeholder="Selectionner un membre" />
    </SelectTrigger>
    <SelectContent>
      {colocation?.members?.map(member => (
        <SelectItem key={member.id} value={member.id}>
          {member.name}
        </SelectItem>
      ))}
    </SelectContent>
  </Select>
  {formErrors.paidBy && <p className="text-sm text-red-500 mt-1">{formErrors.paidBy}</p>}
</div>
*/}

{/* APRES: */}
<div className="space-y-2">
  <Label htmlFor="paidBy">Paye par</Label>
  <select
    id="paidBy"
    value={formData.paidBy}
    onChange={(e) => setFormData({ ...formData, paidBy: e.target.value })}
    className="flex h-8 w-full items-center rounded-lg border border-input bg-transparent px-2.5 py-1 text-sm transition-all outline-none focus-visible:border-ring focus-visible:ring-3 focus-visible:ring-ring/50"
    required
  >
    <option value="">Selectionner un membre</option>
    {colocation?.members?.map(member => (
      <option key={member.id} value={member.id}>
        {member.name}
      </option>
    ))}
  </select>
  {formErrors.paidBy && <p className="text-sm text-red-500 mt-1">{formErrors.paidBy}</p>}
</div>
```

Les classes CSS reproduisent le style du SelectTrigger de `select.jsx` (line 17-19) : `h-8`, `w-full`, `rounded-lg`, `border border-input`, `bg-transparent`, `px-2.5`, `py-1`, `text-sm`.

### Task 2 : Verifier la validation du formulaire

**Fichier : `client/src/pages/Finances.jsx`**

La validation existante (lines 262-264) verifie deja que `formData.paidBy` est non-vide et correspond a un membre :

```javascript
if (!formData.paidBy) newErrors.paidBy = "Le payeur est requis";
else if (!colocation?.members?.some(m => m.id === formData.paidBy)) {

```

Ce code fonctionne identiquement avec le select natif car `formData.paidBy` contient toujours l'`id` du membre (la `value` de l'`<option>`).
