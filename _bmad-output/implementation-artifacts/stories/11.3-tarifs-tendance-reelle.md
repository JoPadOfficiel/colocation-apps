# Story 11.3 : Mise a jour des tarifs et tendance reelle

## User Story

As a **colocataire**,
I want **que les tarifs (montants cagnotte, dettes, creances) se mettent a jour en temps reel apres chaque operation et que la tendance soit basee sur des donnees reelles**,
So that **les chiffres affiches correspondent toujours a la realite**.

## Description

Actuellement dans `Finances.jsx`, la tendance est calculee avec une formule fictive :

```javascript
// Line 80 - Finances.jsx
const tendanceCalculee = cagnotteActuelle * 0.18; // 18% de la cagnotte comme approximation
```

Cela affiche toujours "+44,19 EUR ce mois" quelle que soit l'activite reelle. La tendance doit comparer les depenses du mois courant avec celles du mois precedent pour afficher une variation reelle.

De plus, le graphique `FinancesChart` fonctionne correctement en theorie (il groupe par mois), mais les donnees mockees ne couvrent que mars 2026. Il faut ajouter des donnees sur plusieurs mois pour que le graphique soit utile en demo.

Enfin, le graphique `ExpenseChart` du Dashboard (defini inline dans `Dashboard.jsx` lines 37-78) groupe par `type` (shopping, rent, utility, entertainment) et non par mois, ce qui est different du graphique Finances. Le graphique du Dashboard doit aussi refleter les donnees reelles multi-mois.

## Criteres d'Acceptation

**Given** j'ajoute une depense de 60 EUR partagee (3 colocataires)
**When** la page Finances se rafraichit
**Then** "Mes dettes" ou "On me doit" est recalcule : chaque autre membre doit 20 EUR au payeur
**And** la card "Cagnotte Commune" affiche le montant correct
**And** la tendance affiche la variation reelle par rapport au mois precedent (pas un % fictif)

**Given** les depenses couvrent plusieurs mois (janvier, fevrier, mars...)
**When** je consulte le graphique finances
**Then** le graphique affiche une barre par mois avec les montants reels
**And** je peux voir l'historique complet, pas uniquement le mois courant

**Given** aucune depense n'existe le mois precedent
**When** je consulte la tendance
**Then** la tendance affiche "Nouveau" ou "+100%" au lieu d'un calcul impossible

```javascript
// AVANT (line 78-81):
// const tendanceCalculee = cagnotteActuelle * 0.18;
// setTendance(tendanceCalculee);

// APRES: Calcul reel de la tendance (depenses mois courant vs mois precedent)
const now = new Date();
const currentMonth = now.getMonth();
const currentYear = now.getFullYear();

const depensesMoisCourant = financesData
  .filter(f => f.type !== 'contribution')
  .filter(f => {
    const d = new Date(f.date);
    return d.getMonth() === currentMonth && d.getFullYear() === currentYear;
  })
  .reduce((sum, f) => sum + (f.amount || 0), 0);

const prevMonth = currentMonth === 0 ? 11 : currentMonth - 1;
const prevYear = currentMonth === 0 ? currentYear - 1 : currentYear;

const depensesMoisPrecedent = financesData
  .filter(f => f.type !== 'contribution')
  .filter(f => {
    const d = new Date(f.date);
    return d.getMonth() === prevMonth && d.getFullYear() === prevYear;
  })
  .reduce((sum, f) => sum + (f.amount || 0), 0);

// Tendance = difference entre mois courant et mois precedent
const tendanceCalculee = depensesMoisCourant - depensesMoisPrecedent;
setTendance(tendanceCalculee);
```

### Task 5 : Mettre a jour le ExpenseChart du Dashboard pour le multi-mois

**Fichier : `client/src/pages/Dashboard.jsx`**

Le `ExpenseChart` (lines 37-78) groupe par `type` et non par mois. Ajouter un groupement par mois pour plus de pertinence :

Remplacer la logique interne du ExpenseChart (lines 38-43) par un groupement multi-mois :

```javascript
function ExpenseChart({ finances }) {
  // Group by month instead of type
  const byMonth = {};
  finances.forEach((f) => {
    const date = new Date(f.date);
    if (isNaN(date.getTime())) return;
    const monthKey = `${date.getFullYear()}-${String(date.getMonth() + 1).padStart(2, '0')}`;
    const monthLabel = date.toLocaleDateString("fr-FR", { month: "short" });
    if (!byMonth[monthKey]) {
      byMonth[monthKey] = { label: monthLabel, amount: 0 };
    }
    byMonth[monthKey].amount += f.amount;
  });

  const entries = Object.keys(byMonth)
    .sort()
    .slice(-6)
    .map(key => [byMonth[key].label, byMonth[key].amount]);

  const max = Math.max(...entries.map(([, v]) => v), 1);

  return (
    <Card>
      <CardHeader>
        <CardTitle className="flex items-center gap-2 text-lg">
          <TrendingUp className="w-5 h-5 text-blue-600" />
          Depenses mensuelles
        </CardTitle>
      </CardHeader>
      <CardContent>
        <div className="space-y-3">
          {entries.map(([month, amount]) => (
            <div key={month} className="space-y-1">
              <div className="flex justify-between text-sm">
                <span className="text-gray-600 capitalize">{month}</span>
                <span className="font-medium">{amount.toFixed(2)} EUR</span>
              </div>
              <div className="h-2 bg-gray-100 rounded-full overflow-hidden">
                <div
                  className="h-full rounded-full bg-blue-500"
                  style={{ width: `${(amount / max) * 100}%` }}
                />
              </div>
            </div>
          ))}
        </div>
        <p className="text-sm text-gray-500 mt-4">
          Total : {finances.reduce((sum, f) => sum + f.amount, 0).toFixed(2)} EUR
        </p>
      </CardContent>
    </Card>
  );
}
```
