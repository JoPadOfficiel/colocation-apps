# Story 11.2 : Distinction depenses partagees vs personnelles

## User Story

As a **colocataire**,
I want **distinguer clairement les depenses partagees (divisees entre tous) des depenses personnelles**,
So that **l'equilibre financier est calcule correctement et je sais ce que je dois a qui**.

## Description

Actuellement, toutes les depenses sont traitees de maniere identique : chaque depense ajoutee est divisee entre tous les membres. Le modele `Finance` dans `mockData.js` a un champ `type` utilise pour la categorie de la depense (`shopping`, `rent`, `utility`, `entertainment`), mais rien n'indique si la depense est partagee ou personnelle.

Cette story ajoute un champ `shared` (boolean) au modele Finance et un selecteur "Partagee / Personnelle" dans le formulaire d'ajout/modification de depense. L'equilibre financier ne prend en compte que les depenses `shared: true`.

Le champ `type` existant (`shopping`, `rent`, `utility`, `entertainment`) reste pour la categorisation. Le nouveau champ `shared` est orthogonal.

## Criteres d'Acceptation

**Given** j'ajoute une depense
**When** je choisis "Depense partagee"
**Then** le montant est divise equitablement entre tous les membres de la colocation
**And** l'equilibre de chaque membre est recalcule

**Given** j'ajoute une depense
**When** je choisis "Depense personnelle"
**Then** le montant n'affecte que mon solde personnel
**And** l'equilibre des autres membres n'est pas impacte

**Given** je consulte le tableau des depenses
**When** je regarde la colonne "Type"
**Then** chaque depense affiche un badge "Partagee" ou "Personnelle"

**Given** j'edite une depense existante
**When** le Dialog s'ouvre
**Then** le selecteur affiche la valeur actuelle (partagee ou personnelle)

## Changements API

Aucune nouvelle route. Le champ `shared` est simplement ajoute au body des `POST /api/finances` et `PUT /api/finances/:id` existants.

| Methode | Route | Body modifie | Description |
|---------|-------|-------------|-------------|
| `POST` | `/api/finances` | `{ ..., shared: boolean }` | Ajouter le champ shared |
| `PUT` | `/api/finances/:id` | `{ ..., shared: boolean }` | Mettre a jour le champ shared |
