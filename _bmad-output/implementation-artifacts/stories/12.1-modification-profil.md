# Story 12.1 : Modification complete du profil

## Description

La page Reglages permet actuellement de modifier le nom et l'email, mais il manque la modification du mot de passe. Le formulaire existant fonctionne mais ne met pas correctement a jour le profil affiche dans la sidebar. Cette story complete le profil avec le changement de mot de passe et corrige la synchronisation AuthContext/sessionStorage/sidebar.

## User Story

As a **colocataire**,
I want **modifier mon nom, email et mot de passe depuis les Reglages**,
So that **mes informations personnelles sont toujours a jour**.

## Criteres d'Acceptation

**AC-1: Modification nom/email**
**Given** je suis sur `/settings`
**When** je modifie mon nom et/ou email et clique "Mettre a jour"
**Then** mes informations sont sauvegardees cote serveur
**And** le profil dans la sidebar/header se met a jour immediatement
**And** un message de confirmation vert s'affiche "Informations mises a jour."

**AC-2: Validation nom vide**
**Given** je suis sur `/settings`
**When** je vide le champ nom et clique "Mettre a jour"
**Then** un message d'erreur rouge s'affiche "Le nom ne peut pas etre vide."
**And** aucun appel API n'est envoye

**AC-3: Changement de mot de passe**
**Given** je suis sur `/settings`
**When** je saisis l'ancien mot de passe, le nouveau (>=8 caracteres) et la confirmation
**Then** mon mot de passe est mis a jour cote serveur
**And** un message de confirmation s'affiche

**AC-4: Ancien mot de passe incorrect**
**Given** je tente de changer mon mot de passe
**When** l'ancien mot de passe saisi est incorrect
**Then** un message d'erreur s'affiche "Ancien mot de passe incorrect"

**AC-5: Nouveau mot de passe trop court**
**Given** je tente de changer mon mot de passe
**When** le nouveau mot de passe fait moins de 8 caracteres
**Then** un message d'erreur s'affiche "Le mot de passe doit faire au moins 8 caracteres"

**AC-6: Confirmation mot de passe ne correspond pas**
**Given** je tente de changer mon mot de passe
**When** le champ confirmation ne correspond pas au nouveau mot de passe
**Then** un message d'erreur s'affiche "Les mots de passe ne correspondent pas"
**And** la validation est cote client (pas d'appel API)

## Taches Techniques

### Task 1: Ajouter le formulaire de changement de mot de passe dans Settings.jsx

**File:** `client/src/pages/Settings.jsx`

- Ajouter un nouveau `<Card>` section "Securite" apres la card "Profil" (ligne ~192)
- 3 champs: "Ancien mot de passe", "Nouveau mot de passe", "Confirmer le nouveau mot de passe"
- Utiliser `<Input type="password" />` de shadcn/ui pour chaque champ
- Bouton "Changer le mot de passe" (`<Button>`)
- State: `oldPassword`, `newPassword`, `confirmPassword`, `passwordSaving`, `passwordSuccess`, `passwordError`
- Validation cote client avant appel API:
  - `newPassword.length >= 8`
  - `newPassword === confirmPassword`
- Icone `Lock` de lucide-react pour la section

### Task 2: Modifier la route PUT /api/users/:id pour accepter le mot de passe

**File:** `server/index.js`

La route actuelle (ligne 106-113) exclut `password` du body avec destructuring:
```javascript
const { id, password, role, ...allowed } = req.body;
```

Modifier pour accepter un changement de mot de passe conditionnel:
- Si `req.body.currentPassword` et `req.body.newPassword` sont fournis:
  - Verifier que `users[idx].password === req.body.currentPassword`
  - Si incorrect: retourner `400 { error: "Ancien mot de passe incorrect" }`
  - Si `newPassword.length < 8`: retourner `400 { error: "Le mot de passe doit faire au moins 8 caracteres" }`
  - Sinon: mettre a jour `users[idx].password = req.body.newPassword`
- Continuer a exclure `password` du stripPassword pour la reponse

### Task 3: Ajouter la fonction API client pour le changement de mot de passe

**File:** `client/src/lib/api.js`

- Ajouter `export function changePassword(userId, currentPassword, newPassword)`:
  ```javascript
  return request(`/api/users/${userId}`, {
    method: "PUT",
    body: JSON.stringify({ currentPassword, newPassword })
  })
  ```
- Reutilise la fonction `updateUser` existante ou en cree une dediee

### Task 4: Synchroniser le profil dans la sidebar apres modification

**File:** `client/src/pages/Settings.jsx`

Le code actuel (lignes 122-134) met deja a jour `setUser(updated)` et `sessionStorage`. Verifier que:
- La sidebar (`client/src/components/layout/Sidebar.jsx` ou equivalent) lit `user.name` depuis `useAuth()`
- Le re-render est bien declenche apres `setUser(updated)`
- Si la sidebar cache le nom, forcer le rafraichissement

## Endpoints API

### PUT /api/users/:id (modification)

**Request Body (profil):**
```json
{ "name": "Thomas Durand", "email": "thomas@coloc.fr" }
```

**Request Body (mot de passe):**
```json
{ "currentPassword": "password123", "newPassword": "newpass123" }
```

**Response 200:**
```json
{ "data": { "id": "user-1", "name": "Thomas Durand", "email": "thomas@coloc.fr", "role": "admin", "colocationId": "coloc-1" } }
```

**Response 400 (mauvais ancien mot de passe):**
```json
{ "error": "Ancien mot de passe incorrect" }
```
