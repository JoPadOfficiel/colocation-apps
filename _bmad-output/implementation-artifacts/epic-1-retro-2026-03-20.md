## Problèmes identifiés et corrigés

## Leçons apprises

1. **Toujours nettoyer le boilerplate** après scaffolding (Vite, CRA, etc.)
2. **Les code reviews multi-couches** (Blind Hunter + Edge Case + Acceptance Auditor) sont efficaces — chaque couche trouve des choses différentes
3. **Filtrer les données sensibles** dès la première implémentation, même pour du mock

- Pas de test framework configuré (ni Vitest ni Jest)
- Pas de route guard / protected routes (prévu pour Epic 2)

## Actions pour les prochains Epics
